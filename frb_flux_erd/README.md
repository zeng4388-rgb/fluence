# frblf_erd (Flux Function Version)
A Bayesian framework to measure the event rate density of FRB **flux** function, adapted from the original luminosity function code.

## References

If you build upon this modified framework, please refer to the original works:
- [Luo et al. 2020, MNRAS, 494, 665](https://ui.adsabs.harvard.edu/abs/2020MNRAS.tmp..667L/abstract)
- [Luo et al. 2018, MNRAS, 481, 2320](https://ui.adsabs.harvard.edu/abs/2018MNRAS.481.2320L/abstract)

## Dependencies

- **Python**: 2.7.x (due to print statements)
- **Numpy**: (1.14 at least)
- **Scipy**: (1.0.0 at least)
- **PyMultiNest**: A Python interface to MultiNest
- **Matplotlib**: For plotting the posterior distribution contours.
- **Astropy**: Needed by convert_chime.py to read fits catalogs.
- **MPI Wrapper**: mpiexec or mpirun to implement parallel sampling on MultiNest.

## Workflow & Execution Order

The updated workflow relies on processing real catalog data (e.g., from CHIME) using the Flux function model:

### 1. Data Preparation
Convert the CHIME FITS catalogs into readable .txt components (chime_frb_cat.txt and chime_tel_svy.txt).
`ash
python convert_chime.py
`

### 2. Measure the Quantified FRB Flux Function
Use MultiNest via MPI to compute the posterior distributions of the parameters of the newly modified Flux function. *(Run this in an environment where MPI is configured).*
`ash
./run_samp_flux.sh
`
> **Notes:** The posterior output chains and nested properties will be saved in ./nest_out/samp/.

### 3. Plotting the Posteriors
Read the sampled data chains generated by MultiNest and plot the 1D & 2D posterior distribution contours.
`ash
./draw_samp_flux.sh
`
> **Notes:** The result plots are generated in EPS format and saved in ./plots/samp/.
